# Prompt: App Maintenance

## Scopo

Eseguire revisione profonda dell'app Next.js e della documentazione annessa fuori `documentation/`.

## Scope consentito

- `app/`, `server/`, `shared/`, file root (`README.md`, `HOW_TO.md`, `CONTRIBUTING.md`, `CHANGELOG.md`, ecc.);
- verifica flussi frontend/backend e allineamento con schema Fase 1.

## Scope vietato

- non modificare file dentro `documentation/`.

## Obiettivi operativi

1. trovare bug funzionali, implementativi e logici;
2. ridurre ridondanze/duplicazioni;
3. allineare copy, route e UX;
4. ottimizzare codice senza alterare logica critica del Download.

## Procedura obbligatoria

1. analisi route e componenti principali;
2. audit API/form/schema parity;
3. piano fix con priorita;
4. implementazione incrementale;
5. verifica con lint/build;
6. report finale cambiamenti e rischi residui.

## Output atteso

- app piu stabile e coerente;
- documentazione annessa aggiornata (fuori `documentation/`);
- evidenza verifiche tecniche eseguite.
