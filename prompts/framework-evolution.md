# Prompt: Framework Evolution

## Scopo

Evolvere il framework agendo esclusivamente sulla documentazione in `documentation/`.

## Scope consentito

- modifica, estensione, rifactor e ottimizzazione di file in `documentation/`.

## Scope vietato

- non modificare codice app/backend;
- non alterare file fuori `documentation/`.

## Obiettivi operativi

1. introdurre nuovi tool o regole nelle fasi del workflow (1/2/3);
2. migliorare struttura e leggibilita dei documenti;
3. garantire coerenza metodologica globale;
4. preservare significato e logiche canoniche SMART.

## Procedura obbligatoria

1. definisci impatto del cambiamento sulle fasi;
2. pianifica file da aggiornare;
3. implementa modifiche con allineamento incrociato;
4. aggiorna riferimenti e gate dove necessario;
5. rilascia riepilogo finale dei cambiamenti.
